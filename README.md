# Double Pendulum Encryption

**A chaos-based encryption system using double pendulums**

[![Live Demo](https://img.shields.io/badge/Live-Demo-brightgreen)](https://akingry.github.io/pendulum-cipher/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

## Overview

Double Pendulum Encryption is a cryptographic tool that uses deterministic chaos for encryption. It simulates two double pendulums to generate a cryptographically-secure keystream, which is then XORed with plaintext to produce ciphertext.

---

## How It Works

### The Double Pendulum: A Chaotic System

A double pendulum consists of one pendulum attached to the end of another. Despite being governed by deterministic physics (Newton's laws), double pendulums exhibit **chaotic behavior**:

- **Sensitive dependence on initial conditions**: Tiny differences in starting position lead to wildly different trajectories
- **Deterministic but unpredictable**: Given exact initial conditions, the motion is perfectly reproducible, but practically impossible to predict long-term

### Physics: Lagrangian Mechanics

The motion is computed using the Lagrangian formulation of classical mechanics:

```
L = T - V (Lagrangian = Kinetic Energy - Potential Energy)
```

The equations of motion are derived from the Euler-Lagrange equations, resulting in two coupled second-order differential equations. These are solved numerically using **RK4 (Runge-Kutta 4th order)** integration.

## Encryption Process

### Step 1: Key Derivation

Your password/key is converted into pendulum initial conditions:

```
Password -> SHA-512 -> 64 bytes -> 16 parameters
```

These 16 parameters determine:
- **Pendulum 1**: L1, L2, m1, m2, theta1, theta2, omega1, omega2
- **Pendulum 2**: L1, L2, m1, m2, theta1, theta2, omega1, omega2

Same key = Same initial conditions = Same chaotic trajectory = Same keystream

### Step 2: Chaos Steps (Warmup)

Before generating the keystream, both pendulums are simulated for a configurable number of steps (500-2000). This allows the chaos to fully develop before extracting the keystream.

**Important:** The number of chaos steps affects the final keystream. Different step counts produce different ciphertexts, even with the same key! You must use the same step count for encryption and decryption.

### Step 3: Keystream Generation

After the chaos steps complete, the keystream is generated by repeatedly:

1. Advancing the simulation by one timestep (dt = 0.001s)
2. Collecting the state of both pendulums (positions, velocities)
3. Including the elapsed simulation time
4. Hashing everything with SHA-256 to produce 32 bytes

This process repeats until enough keystream bytes are generated for the plaintext.

### Step 4: Output Encoding

The ciphertext can be encoded in two formats:

- **Base64** - compact, URL-safe encoding (default)
- **Words (BIP39)** - human-readable mnemonic words, easier to transcribe or speak aloud

### Step 5: XOR Encryption

The plaintext is XORed with the keystream to produce ciphertext.

### Step 6: Decryption

XOR is its own inverse. To decrypt:

1. Use the **same key** (same initial conditions)
2. Use the **same chaos steps** (same warmup)
3. Generate the **identical keystream**
4. XOR with ciphertext to recover plaintext
